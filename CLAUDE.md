# System Prompt

Проект строится по фреймворку **DOE** (Directions, Orchestration, Execution).

## DOE Framework

### Directions — регламенты и SOP
Стандартные операционные процедуры (SOP), правила, ограничения, цели. Аналог регламентов: что делать, чего не делать, какие стандарты соблюдать. Это статичные документы, которые задают рамки.

Файлы: `directions/` — markdown-документы с требованиями, правилами, контекстом.

### Orchestration — LLM как оркестратор
Роль LLM: читает directions, анализирует задачу, строит план, распределяет шаги. LLM не пишет финальный код — он координирует и принимает решения. Аналог менеджера/архитектора.

Файлы: `orchestration/` — планы, декомпозиция задач, промпты.

### Execution — скрипты и код
Непосредственное выполнение: скрипты, вызовы API, генерация кода, запуск команд. Это инструменты, которые оркестратор вызывает по плану.

Файлы: `execution/` — код, скрипты, конфиги, результаты.

## Структура проекта

```
directions/       # SOP, регламенты, требования, ограничения
orchestration/    # Планы, декомпозиция, координация (LLM)
execution/        # Скрипты, код, результаты выполнения
```

## Правила

- Перед началом работы — определи Directions.
- Перед написанием кода — составь Orchestration-план.
- Execution — только после утверждённого плана.
